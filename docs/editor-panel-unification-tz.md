# ТЗ: Унификация редакторов кода в Hub

## 1. Контекст и проблема

### 1.1. Текущее состояние
В проекте Hub существует несколько редакторов кода, которые визуально и архитектурно различаются:

1. **`<JsonEditor>`** — используется в Command Tester и System Configuration
2. **`<CSharpEditor>`** — используется в Configurator для редактирования стейджей процессов
3. **`<BuildRulesEditor>`** — используется в System Configuration для редактирования секций адаптеров
4. **`<GlobalModelEditor>`** — используется в Configurator для редактирования глобальных моделей
5. **Прямое использование `<CodeEditor>`** — в различных местах

### 1.2. Проблемы
- **Визуальная несогласованность**: Разные отступы, размеры шрифтов, регистр текста в заголовках
- **Архитектурное дублирование**: Каждый редактор создает свои заголовки и тулбары
- **Сложность поддержки**: Изменение стиля требует правки в 5+ местах
- **Отсутствие единого API**: Разные пропсы и поведение у похожих компонентов

## 2. Цель

Создать унифицированный компонент `<EditorPanel>`, который будет использоваться всеми редакторами кода в проекте, обеспечивая:

1. Единый визуальный стиль
2. Консистентный UX
3. Упрощенную поддержку и расширяемость
4. Поддержку стандартных действий (Save, Validate, Format и др.)

## 3. Требования к компоненту

### 3.1. Базовые требования
- **Расположение**: `src/components/ui/EditorPanel/`
- **Зависимости**: Должен использовать существующие UI-kit компоненты (`<PanelHeader>`, `<PanelToolbar>`, `<CodeEditor>`)
- **Типизация**: Полная TypeScript типизация
- **Доступность**: Поддержка ARIA атрибутов и клавиатурной навигации

### 3.2. Функциональные требования

#### 3.2.1. Заголовок панели
- Поддержка иконки слева
- Текст заголовка (обязательно)
- Опциональный бейдж (например, счетчик ошибок)
- Опциональная подсказка (hint) - дополнительная информация
- Группа действий справа (кнопки)

#### 3.2.2. Редактор кода
- Поддержка всех языков из `<CodeEditor>` (C#, JSON, JavaScript, TypeScript и др.)
- Автоматический выбор темы по языку (wfm-dark для C#, hub-dark для остальных)
- Поддержка маркеров ошибок/предупреждений
- Настройки word wrap, minimap, folding
- Горячие клавиши редактора

#### 3.2.3. Стандартные действия
- **Save** (Ctrl+S) — сохранение
- **Validate** (Ctrl+Shift+V) — валидация
- **Format** (Shift+Alt+F) — форматирование
- **Run** (F5) — запуск (опционально)
- **Toggle Problems** (Ctrl+Shift+M) — переключение панели проблем

#### 3.2.4. Состояния
- **Normal** — обычное состояние
- **Dirty** — есть несохраненные изменения (визуальный индикатор)
- **Saving** — сохранение в процессе
- **Validating** — валидация в процессе
- **Error** — ошибка валидации/сохранения

### 3.3. API компонента

```typescript
interface EditorPanelProps {
  // === Заголовок ===
  title: string;
  icon?: ReactNode;
  badge?: ReactNode;
  hint?: string;
  
  // === Действия ===
  actions?: Array<{
    id: string;
    label: string;
    icon: ReactNode;
    shortcut?: string;
    disabled?: boolean;
    variant?: 'primary' | 'secondary' | 'danger';
    onClick: () => void;
  }>;
  
  // === Редактор ===
  value: string;
  onChange?: (value: string) => void;
  language: CodeEditorLanguage;
  readOnly?: boolean;
  
  // === Состояние ===
  state?: {
    dirty?: boolean;
    saving?: boolean;
    validating?: boolean;
    error?: string;
  };
  
  // === Маркеры ===
  markers?: CodeEditorMarker[];
  
  // === Внешние зависимости ===
  beforeMount?: (monaco: Monaco) => void;
  onMount?: (editor: MonacoNs.editor.IStandaloneCodeEditor, monaco: Monaco) => void;
  
  // === Стилизация ===
  variant?: 'default' | 'compact' | 'borderless';
  className?: string;
  style?: CSSProperties;
  
  // === Дополнительные опции ===
  showHeader?: boolean;
  showToolbar?: boolean;
  height?: string | number;
  minimap?: boolean;
  wordWrap?: 'off' | 'on' | 'bounded';
  
  // === Пропсы для вложенных компонентов ===
  headerProps?: Partial<PanelHeaderProps>;
  editorProps?: Partial<CodeEditorProps>;
}
```

## 4. Архитектурные решения

### 4.1. Структура компонента
```
<EditorPanel>
  ├── <PanelHeader> (если showHeader=true)
  │   ├── Иконка
  │   ├── Заголовок + hint
  │   ├── Бейдж
  │   └── Действия (кнопки)
  ├── <PanelToolbar> (если showToolbar=true, опционально)
  │   ├── Поиск/фильтры слева
  │   └── Дополнительные действия справа
  └── <CodeEditor>
```

### 4.2. Интеграция с существующими компонентами
- **Использовать существующие**: `<PanelHeader>`, `<PanelToolbar>`, `<CodeEditor>`
- **Не дублировать**: Не создавать новую логику заголовков/тулбаров
- **Расширять при необходимости**: Добавлять только недостающую функциональность

### 4.3. Поддержка специализированных редакторов
- **WFM C#**: Поддержка `beforeMount` для `setupWfmCSharp`
- **JSON**: Автоматическое форматирование и валидация
- **Build Rules**: Комплексный редактор с двумя панелями
